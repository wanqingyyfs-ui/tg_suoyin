# rectg 项目使用说明

## 📋 项目概述

**rectg** 是一个 **Telegram 中文频道/群组搜索引擎项目**，基于 SQLite、Python 爬虫和 Astro 前端构建。

- **项目名称**: rectg (Telegram Resource Index)
- **项目仓库**: https://github.com/wanqingyyfs-ui/rectg
- **本地路径**: `D:\编程\wanqing\projects\rectg`
- **数据来源**: Telegram 公开页面爬虫
- **数据存储**: SQLite 数据库 (`data/rectg.db`)
- **前端技术**: Astro 静态站点生成器
- **部署方式**: 静态网站（支持 Vercel/Cloudflare Pages）

---

## 🎯 项目功能

### 核心功能
1. **数据爬取**: 从 Telegram 公开页面抓取频道/群组信息
2. **数据清洗**: 自动清洗标题和简介，自动分类
3. **数据管理**: 支持手动编辑分类、状态、广告位
4. **搜索引擎**: 提供前端搜索和 Bot 搜索功能
5. **静态站点**: 生成高性能的静态搜索站点
6. **广告系统**: 支持多位置广告管理和展示

### 已实现功能
- ✅ SQLite 数据库管理
- ✅ Python 爬虫系统
- ✅ 自动分类系统
- ✅ 前端静态站点
- ✅ 搜索功能（前端+Bot）

- ✅ 后台管理界面

## 📊 数据库结构

### 主数据库: `data/rectg.db`

#### 1. `links` 表 - 待抓取链接
```sql
CREATE TABLE links (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    url TEXT NOT NULL UNIQUE,
    username TEXT,
    name TEXT,
    type_hint TEXT,  -- channel / group / bot
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. `entries` 表 - 已抓取数据
```sql
CREATE TABLE entries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    telegram_id TEXT,
    username TEXT UNIQUE NOT NULL,
    url TEXT NOT NULL,
    type TEXT,  -- channel / group / bot
    title TEXT,
    description TEXT,
    clean_title TEXT,
    clean_desc TEXT,
    category TEXT,  -- 分类标签，如 "💻 数码科技"
    avatar TEXT,
    count INTEGER,
    last_active TEXT,
    valid INTEGER DEFAULT 1,      -- 是否有效 0/1
    private INTEGER DEFAULT 0,    -- 是否私密 0/1
    keep INTEGER DEFAULT 0,       -- 是否展示 0/1
    filter_reason TEXT,           -- 过滤原因
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

前端只展示满足以下条件的数据：
```sql
keep = 1 AND valid = 1 AND private = 0
```

#### 3. `ads` 表 - 广告位
```sql
CREATE TABLE ads (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    position TEXT NOT NULL,    -- home_top / home_sidebar / bot_search_inline
    title TEXT NOT NULL,
    description TEXT,
    url TEXT,
    image_url TEXT,
    sort_order INTEGER DEFAULT 0,
    enabled INTEGER DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 📝 常用操作

### 1. 添加新频道/群组

**方式 A: 只添加链接**
```powershell
python scripts\add_entry.py https://t.me/your_username --type channel
```

**方式 B: 添加并立即爬取**
```powershell
python scripts\add_entry.py your_username --type channel --crawl
```

**方式 C: 强制展示（人工审核后）**
```powershell
python scripts\add_entry.py your_username --keep
```

**参数说明**:
- `--type`: channel（频道）/ group（群组）/ bot（机器人）
- `--crawl`: 添加后立即爬取
- `--keep`: 强制设置为可展示状态

### 2. 爬取数据

**爬取新链接**（推荐）:
```powershell
python scripts\crawl.py --new --no-active
```

**测试爬取前 5 条**:
```powershell
python scripts\crawl.py --limit 5 --no-active
```

**全量重爬**（慎用）:
```powershell
python scripts\crawl.py --no-resume --no-active
```

### 3. 自动清洗和分类

```powershell
python scripts\categorize.py
```

功能：
- 重新评估 `keep` / `filter_reason`
- 清洗标题和简介
- 自动分类到预设类别

### 4. 手动编辑分类

**查看分类统计**:
```powershell
python scripts\edit_category.py stats
```

**查看内置分类列表**:
```powershell
python scripts\edit_category.py categories
```

**查看某个频道的分类**:
```powershell
python scripts\edit_category.py get your_username
```

**修改分类**:
```powershell
python scripts\edit_category.py set your_username "💻 数码科技"
```

**查看未分类数据**:
```powershell
python scripts\edit_category.py list --uncategorized --limit 30
```

**允许创建新分类**:
```powershell
python scripts\edit_category.py set your_username "新分类" --allow-new
```

### 5. 手动编辑状态

**查看状态**:
```powershell
python scripts\edit_status.py get your_username
```

**手动放行（展示）**:
```powershell
python scripts\edit_status.py set your_username --keep 1 --valid 1 --private 0 --clear-reason
```

**手动隐藏**:
```powershell
python scripts\edit_status.py set your_username --keep 0 --reason "人工隐藏"
```

**查看被过滤的数据**:
```powershell
python scripts\edit_status.py list --filtered --limit 30
```

**查看私密数据**:
```powershell
python scripts\edit_status.py list --private --limit 30
```

### 6. 广告位管理

**初始化广告表**:
```powershell
python scripts\manage_ads.py init
```

**添加广告**:
```powershell
python scripts\manage_ads.py add home_top "首页顶部广告" --url "https://example.com" --description "广告说明"
```

**添加但默认禁用**:
```powershell
python scripts\manage_ads.py add home_top "测试广告" --url "https://example.com" --disabled
```

**查看启用的广告**:
```powershell
python scripts\manage_ads.py list
```

**查看全部广告**:
```powershell
python scripts\manage_ads.py list --all
```

**启用/禁用广告**:
```powershell
python scripts\manage_ads.py enable 1
python scripts\manage_ads.py disable 1
```

**更新广告**:
```powershell
python scripts\manage_ads.py update 1 --title "新标题" --sort-order 10
```

**删除广告**:
```powershell
python scripts\manage_ads.py delete 1 --yes
```

**广告位 position 选项**:
- `home_top`: 首页顶部
- `home_sidebar`: 首页侧边栏
- `home_below_hero`: 首页英雄区下方
- `bot_search_inline`: Bot 搜索结果内嵌

## ⚠️ 注意事项

### 1. PowerShell 长脚本问题

PowerShell 的 `PSReadLine` 在粘贴超长命令时可能崩溃，建议：
- 不要在终端粘贴过长的 Python 代码
- 优先使用短命令测试
- 复杂脚本应写入 `.py` 文件再执行

### 2. Emoji 分类输入

某些终端可能无法正确显示 Emoji。如果遇到问题：
```powershell
# 先查看内置分类列表
python scripts\edit_category.py categories

# 如果 Emoji 显示异常，可以先用 ASCII 测试
python scripts\edit_category.py set username TEST_CATEGORY --allow-new
```

### 3. Git 不提交的文件

`.gitignore` 已配置，不要提交以下内容：
```
.venv/
node_modules/
web/node_modules/
web/dist/
.astro/
web/.astro/
.env
.env.*
*.log
*.bak
*.backup
*.tmp
```

### 4. 数据库备份

**重要**: 定期备份 `data/rectg.db`！

```powershell
# 手动备份
cp data/rectg.db data/rectg.db.backup

# 或使用时间戳
$timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
cp data/rectg.db "data/rectg.db.$timestamp.backup"
```

### 5. 测试数据恢复

如果在测试时修改了数据，可以用 SQL 恢复：
```powershell
python -c "import sqlite3; conn=sqlite3.connect('data/rectg.db'); conn.execute('UPDATE entries SET category = NULL WHERE username = ?', ('your_username',)); conn.commit(); conn.close(); print('restored')"
```
